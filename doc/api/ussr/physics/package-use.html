Uses of Package
ussr.physics

Packages that use ussr.physics
ussr.comm An abstract communications model comprising transmitters, receivers, and packets. 
ussr.model An abstract model of a modular robot to be used from the controller. 
ussr.physics An abstract model of a physics engine, serves as the interface between physics-engine independent classes and the physics engine. 
ussr.physics.jme A physics simulation implementation based on JME (JMonkeyEngine). 
ussr.physics.jme.actuators Provides a collection of actuators for the JME physics engine implementation. 
ussr.physics.jme.connectors Provides a collection of connectors for the JME physics engine implementation. 
ussr.physics.jme.robots Robot implementations for the ussr.physics.jme simulation implementation. 
ussr.physics.jme.sensors Provides a collection of sensors for the JME physics engine implementation. 
ussr.samples Generic infrastructure for simulations as well as examples of simulations mixing different robots. 
ussr.samples.atron Simulation examples based on the ATRON modular self-reconfigurable robot. 
ussr.samples.atron.natives ATRON examples based on native (implemented in C) controllers 
ussr.samples.atron.network ATRON network controller. 
ussr.samples.atron.simulations Concrete examples of ATRON simulations. 
ussr.samples.mtran Preliminary simulations based on the M-TRAN modular self-reconfigurable robot 
ussr.samples.odin.simulations Concrete Odin simulation examples. 
ussr.samples.white Simple and very preliminary example of a reconfigurable robot with hinge-joints at the edges.
