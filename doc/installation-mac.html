USSR Installation on Mac

Simulations must run using the 32-bit memory model, which requires a the -d32 option to be passed to the JVM. In eclipse, this is done as follows:
  1. Click the menu item "Eclipse->Preferences..."
  2. In the categories on the left click "Java->Installed JREs"
  3. Under installed JREs highlight the one that has a check box next to it (probably JDK 1.6) and click the "Edit..." button
  4. Under "Default VM arguments" add the text "-d32" (without the quotes)
If this option has not been added correctly, starting a simulation will produce an exception similar to the following: 
java.lang.UnsatisfiedLinkError: /Users/ups/src/ussr/zoo/jME_2.0/lib/natives/libodejava.jnilib:  no suitable image found.  Did find:  /Users/ups/src/ussr/zoo/jME_2.0/lib/natives/libodejava.jnilib: no matching architecture in universal wrapper
	at java.lang.ClassLoader$NativeLibrary.load(Native Method)
	...
The simulation will appear to start but nothing moves.
